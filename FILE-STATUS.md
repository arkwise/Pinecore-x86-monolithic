> **Check this before editing any source file.** Each entry records the file's current stability status and a short note about recent changes. The status drives the edit policy below — STABLE files are treated as read-only unless you're fixing a confirmed bug. See [`CONTRIBUTING.md`](CONTRIBUTING.md) §2 for the pre-edit checklist.

---

## Status Definitions

| Status | Meaning | Edit Policy |
|--------|---------|-------------|
| **STABLE** | Complete and tested | Do NOT edit unless fixing a confirmed bug |
| **ACTIVE** | Currently being developed | Edit freely within task scope |
| **EXPERIMENTAL** | Spike/prototype | Edit freely, may be deleted |
| **PLANNED** | Doesn't exist yet | Create when roadmap requires it |
| **FROZEN** | Deliberately locked | Do NOT edit without user instruction |
