# 16550 UART — Full Serial Port Driver

> National Semiconductor 16550A. Standard PC serial port with 16-byte FIFO. We already have basic polled serial for debug; this documents the full IRQ-driven driver with FIFO.

**Date:** 2026-05-02
**Status:** Complete — register-level reference for implementation
**Source:** National Semiconductor PC16550D datasheet

---

## Findings

### Base Addresses and IRQs

| Port | Base | IRQ |
|------|------|-----|
| COM1 | 0x3F8 | IRQ 4 (INT 36) |
| COM2 | 0x2F8 | IRQ 3 (INT 35) |
| COM3 | 0x3E8 | IRQ 4 |
| COM4 | 0x2E8 | IRQ 3 |

### Register Map (8 registers, DLAB multiplexed)

| Offset | DLAB=0 Read | DLAB=0 Write | DLAB=1 | Purpose |
|--------|-------------|--------------|--------|---------|
| +0 | RBR (Rx data) | THR (Tx data) | DLL (divisor low) | Data / baud rate |
| +1 | IER | IER | DLH (divisor high) | Interrupt enable / baud rate |
| +2 | IIR (read) | FCR (write) | — | Interrupt ID / FIFO control |
| +3 | LCR | LCR | — | Line control (data bits, parity, stop) |
| +4 | MCR | MCR | — | Modem control (DTR, RTS, OUT2) |
| +5 | LSR | — | — | Line status (Rx ready, Tx empty, errors) |
| +6 | MSR | — | — | Modem status (CTS, DSR, RI, DCD) |
| +7 | SCR | SCR | — | Scratch register |

### LCR (Line Control Register)

| Bits | Value | Meaning |
|------|-------|---------|
| 1-0 | 00=5, 01=6, 10=7, 11=8 | Data bits |
| 2 | 0=1 stop, 1=2 stop | Stop bits |
| 3 | 0=no parity, 1=parity | Parity enable |
| 4 | 0=odd, 1=even | Parity type |
| 7 | DLAB | Divisor Latch Access Bit |

**Common:** 8N1 = `0x03`, 7E1 = `0x1A`

### Baud Rate Divisors (clock = 1.8432 MHz)

| Baud | Divisor | DLH:DLL |
|------|---------|---------|
| 115200 | 1 | 0x00:0x01 |
| 57600 | 2 | 0x00:0x02 |
| 38400 | 3 | 0x00:0x03 |
| 19200 | 6 | 0x00:0x06 |
| 9600 | 12 | 0x00:0x0C |
| 4800 | 24 | 0x00:0x18 |
| 2400 | 48 | 0x00:0x30 |
| 1200 | 96 | 0x00:0x60 |

### IER (Interrupt Enable)

| Bit | Name | Purpose |
|-----|------|---------|
| 0 | RDA | Rx Data Available |
| 1 | THRE | Tx Holding Register Empty |
| 2 | RLS | Rx Line Status (error) |
| 3 | MS | Modem Status change |

### IIR (Interrupt Identification, read-only)

| Bits 2-1 | Priority | Source | Clear by |
|----------|----------|--------|----------|
| 11 | Highest | Line Status error | Read LSR |
| 10 | High | Rx Data Available | Read RBR |
| 01 | Medium | Tx Holding Empty | Write THR or read IIR |
| 00 | Low | Modem Status | Read MSR |

Bit 0: 0=interrupt pending, 1=no interrupt

### FCR (FIFO Control, write-only)

| Bit | Purpose |
|-----|---------|
| 0 | Enable FIFOs (must be 1 for 16550A) |
| 1 | Clear Rx FIFO (self-clearing) |
| 2 | Clear Tx FIFO (self-clearing) |
| 7-6 | Rx trigger level: 00=1, 01=4, 10=8, 11=14 bytes |

**Common:** `0xC7` = enable FIFO, clear both, trigger at 14 bytes

### LSR (Line Status, read-only)

| Bit | Name | Meaning |
|-----|------|---------|
| 0 | DR | Data Ready (byte in RBR) |
| 1 | OE | Overrun Error |
| 2 | PE | Parity Error |
| 3 | FE | Framing Error |
| 4 | BI | Break Interrupt |
| 5 | THRE | Tx Holding Register Empty |
| 6 | TEMT | Transmitter completely empty |

### MCR (Modem Control)

| Bit | Name | Purpose |
|-----|------|---------|
| 0 | DTR | Data Terminal Ready |
| 1 | RTS | Request To Send |
| 3 | OUT2 | **Must be set to enable IRQs on PC** |
| 4 | LOOP | Loopback mode (for testing) |

### Full Init Sequence (115200 8N1, IRQ-driven)

```
1. outb(base+1, 0x00)     -- disable all interrupts
2. outb(base+3, 0x80)     -- set DLAB
3. outb(base+0, 0x01)     -- divisor low = 1 (115200 baud)
4. outb(base+1, 0x00)     -- divisor high = 0
5. outb(base+3, 0x03)     -- clear DLAB, set 8N1
6. outb(base+2, 0xC7)     -- enable FIFO, clear, 14-byte trigger
7. outb(base+4, 0x0B)     -- MCR: DTR + RTS + OUT2
8. outb(base+1, 0x03)     -- IER: enable RDA + THRE
```

### 16550 Detection

```
outb(base+2, 0xE7)       -- try to enable FIFO with all features
val = inb(base+2)         -- read IIR
if (val & 0xC0) == 0xC0:  16550A (FIFO working)
if (val & 0xC0) == 0x80:  16550 (broken FIFO)
if (val & 0xC0) == 0x00:  8250 or 16450 (no FIFO)
```

### Implementation Notes for Pinecore

1. We already have basic polled COM1 in `serial.c` — this documents the path to full IRQ + FIFO
2. OUT2 (MCR bit 3) is essential — without it, IRQs are gated off on PC hardware
3. The 14-byte trigger level reduces interrupt frequency — good for performance
4. For a DOS modem/BBS experience, need to implement flow control (RTS/CTS via MCR/MSR)
