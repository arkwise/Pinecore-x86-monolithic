# RTL8139 PCI Ethernet NIC

> Most common PCI NIC of the DOS/Win9x era. Simple ring-buffer Rx, 4-descriptor Tx. ~400 lines of C for a complete driver.

**Date:** 2026-05-02
**Status:** Complete — register-level reference for implementation
**Source:** Realtek RTL8139D Datasheet (80 pages)

---

## Findings

### PCI Identification

| Field | Value |
|-------|-------|
| Vendor ID | 0x10EC (Realtek) |
| Device ID | 0x8139 |
| Class | 0x02 (Network) |
| Subclass | 0x00 (Ethernet) |
| BAR0 | I/O base address |
| QEMU flag | `-net nic,model=rtl8139 -net user` |

### Register Map (offsets from I/O base)

#### Identity / MAC
| Offset | Size | Name | R/W | Purpose |
|--------|------|------|-----|---------|
| 0x00-0x05 | 6 | IDR0-5 | RO | MAC address (6 bytes) |

#### Transmit (4 descriptors, round-robin)
| Offset | Size | Name | Purpose |
|--------|------|------|---------|
| 0x10 | 4 | TSD0 | Tx Status Descriptor 0 |
| 0x14 | 4 | TSD1 | Tx Status Descriptor 1 |
| 0x18 | 4 | TSD2 | Tx Status Descriptor 2 |
| 0x1C | 4 | TSD3 | Tx Status Descriptor 3 |
| 0x20 | 4 | TSAD0 | Tx Start Address 0 (physical addr of packet buffer) |
| 0x24 | 4 | TSAD1 | Tx Start Address 1 |
| 0x28 | 4 | TSAD2 | Tx Start Address 2 |
| 0x2C | 4 | TSAD3 | Tx Start Address 3 |

**TSD bit layout:**
| Bits | Purpose |
|------|---------|
| 12-0 | Packet size in bytes |
| 13 | OWN — 1=hardware owns, 0=driver owns |
| 16 | TUN — transmit underrun |
| 17 | TOK — transmit OK |

#### Receive
| Offset | Size | Name | Purpose |
|--------|------|------|---------|
| 0x30 | 4 | RBSTART | Rx buffer physical address (4-byte aligned) |
| 0x38 | 2 | CAPR | Current Address of Packet Read (driver advances) |
| 0x3A | 2 | CBR | Current Buffer Read (hardware write pointer, RO) |

#### Control
| Offset | Size | Name | Purpose |
|--------|------|------|---------|
| 0x37 | 1 | CR | Command: bit 0=Rx enable, 1=Tx enable, 4=Reset |
| 0x3C | 2 | IMR | Interrupt Mask Register |
| 0x3E | 2 | ISR | Interrupt Status Register (write 1 to clear) |
| 0x40 | 4 | TCR | Transmit Configuration |
| 0x44 | 4 | RCR | Receive Configuration |
| 0x52 | 1 | CONFIG1 | Power management (clear bit 7 to power on) |

### IMR/ISR Bits

| Bit | Name | Meaning |
|-----|------|---------|
| 0 | ROK | Receive OK |
| 1 | RER | Receive Error |
| 2 | TOK | Transmit OK |
| 3 | TER | Transmit Error |
| 4 | RXOVW | Rx Buffer Overflow |
| 5 | PUN | Link Change |
| 6 | FOVW | Rx FIFO Overflow |
| 15 | SERR | System Error |

### RCR (Receive Configuration, offset 0x44)

| Bit | Name | Purpose |
|-----|------|---------|
| 0 | AAP | Accept All Packets (promiscuous) |
| 1 | APM | Accept Physical Match (our MAC) |
| 2 | AM | Accept Multicast |
| 3 | AB | Accept Broadcast |
| 4 | AR | Accept Runt (<64 bytes) |
| 6 | WRAP | Wrap ring buffer |
| 12-11 | BUFSZ | Buffer size: 0=8K, 1=16K, 2=32K, 3=64K (+16 bytes always) |

### Initialization Sequence

```
1. Power on:     outb(io + 0x52, 0x00)         -- clear PWRDWN in CONFIG1
2. Reset:        outb(io + 0x37, 0x10)          -- set Reset bit in CR
3. Wait:         while (inb(io+0x37) & 0x10)    -- poll until clear
4. Rx buffer:    outl(io + 0x30, rx_buf_phys)   -- RBSTART = physical addr
5. Init CAPR:    outw(io + 0x38, 0)             -- start of buffer
6. Enable:       outb(io + 0x37, 0x03)          -- CR: Rx + Tx enable
7. RCR:          outl(io + 0x44, 0x0000080E)    -- APM+AB+WRAP+32KB
8. TCR:          outl(io + 0x40, 0x03000000)    -- normal mode
9. IMR:          outw(io + 0x3C, 0x000F)        -- ROK+RER+TOK+TER
10. Read MAC:    for i in 0..5: mac[i] = inb(io + i)
```

### Receiving Packets

Ring buffer in system RAM. Each packet has a 4-byte header:
```
Bytes 0-1: Status (bit 0 = ROK)
Bytes 2-3: Packet length (including 4-byte CRC)
Bytes 4+:  Ethernet frame data
```

Rx loop:
```
while CAPR != CBR:
    header = *(uint32_t *)(rx_buf + (CAPR % buf_size))
    status = header & 0xFFFF
    length = (header >> 16) & 0xFFFF
    if status & 1:  // ROK
        process_packet(rx_buf + CAPR + 4, length - 4)
    CAPR = (CAPR + 4 + length + 3) & ~3   // round up to 4-byte boundary
    outw(io + 0x38, CAPR - 16)            // update CAPR (minus 16 for hardware quirk)
```

### Transmitting Packets

4 descriptors round-robin. To send:
```
1. Copy packet to tx_buf[tx_idx] (physical memory, <1792 bytes)
2. outl(io + 0x20 + tx_idx*4, tx_buf_phys[tx_idx])    -- TSAD = buffer addr
3. outl(io + 0x10 + tx_idx*4, length & 0x1FFF)        -- TSD = size (triggers send)
4. tx_idx = (tx_idx + 1) % 4
```

Check completion: read TSD, check bit 17 (TOK) or bit 16 (TUN for error).

### Interrupt Handler

```
isr = inw(io + 0x3E)           -- read ISR
if isr & 0x01: rx_handler()    -- ROK
if isr & 0x04: tx_complete()   -- TOK
if isr & 0x10: rx_overflow()   -- RXOVW: reset Rx ring
outw(io + 0x3E, isr)           -- acknowledge all by writing back
pic_eoi(irq)
```

### Memory Requirements

| Buffer | Size | Alignment | Constraint |
|--------|------|-----------|------------|
| Rx ring | 32KB + 16 | 4-byte | Must be contiguous physical memory |
| Tx bufs | 4 x 2KB | 4-byte | Each must not cross page boundary |

### Implementation Notes for Pinecore

1. **Prerequisite:** PCI enumeration (ch-20) to find the RTL8139 and read BAR0/IRQ
2. **Physical memory:** Use PMM to allocate contiguous pages for Rx buffer (8 pages = 32KB)
3. **Identity mapping:** Our first 4MB is identity-mapped, so phys == virt for buffers below 4MB
4. **PCI bus master:** Must set command register bit 2 before DMA works
5. **QEMU testing:** `qemu-system-i386 ... -net nic,model=rtl8139 -net user`
6. **No TCP/IP needed initially** — can test with raw Ethernet frames, ARP, or UDP
