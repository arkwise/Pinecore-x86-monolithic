# 3.56 - DOS function 0FF98h - DOS/32 Advanced Map Physical Memory
| **In:** | **AX** = 0FF98h<br> **EBX** = physical address of memory<br> **ESI** = size of region to map |
|---|---|
| **Out:** | <br><br>if successful:<br> **CF** clear<br> **EBX** = linear address that can be used to access physical memory <br><br>if failed:<br> **CF** set |

**Notes: **

a) This call is specific to DOS/32 Advanced DOS Extender only, and is not supported by standard DOS.

b) This function is equivalent to DPMI function 0800h except that it uses 32-bit registers instead of 16:16-bit pair of registers.
